## 1. Executive Summary

A comprehensive supply-chain review was conducted for both the Rust
reference implementation and the Go independent implementation. The
review covers SBOM generation, vulnerability scanning, license review,
reproducibility assessment, and cryptographic dependency analysis.

**Key finding**: The Rust implementation has 4 unmaintained crate
warnings from `cargo audit`, all related to the `pqcrypto` family
(ML-DSA-65 implementation). The upstream PQClean project is being
archived. This is a **HIGH severity finding** that must be addressed
before the release candidate. The Go implementation has zero external
dependencies and zero vulnerabilities.

## 5. Vulnerability Scan

### 5.1 Rust (cargo audit)

- **File**: `supply_chain/cargo-audit.txt`
- **Tool**: cargo-audit 0.22.2
- **Database**: RustSec advisory-db (1138 advisories)
- **Scan date**: 2025-01-15
- **Result**: 0 vulnerabilities, 4 unmaintained warnings

**Unmaintained warnings**:

| Crate | Version | Advisory | Date | Severity |
|-------|---------|----------|------|----------|
| pqcrypto-internals | 0.2.11 | RUSTSEC-2026-0163 | 2026-06-04 | HIGH |
| pqcrypto-mldsa | 0.1.2 | RUSTSEC-2026-0166 | 2026-06-04 | HIGH |
| pqcrypto-traits | 0.3.5 | RUSTSEC-2026-0162 | 2026-06-04 | HIGH |
| paste | 1.0.15 | RUSTSEC-2024-0436 | 2024-10-07 | LOW |

**Analysis**:

The `pqcrypto` family (3 advisories) provides the ML-DSA-65
post-quantum signature implementation used for agent identity and
handshake authentication. The upstream PQClean project is being
archived, meaning these crates will no longer receive security
patches or bug fixes. This is a critical supply-chain risk for a
protocol that depends on post-quantum signatures as its primary
authentication mechanism.

The `paste` crate (1 advisory) is a transitive dependency of
`pqcrypto-internals` used for macro hygiene. It is unmaintained but
poses minimal risk as it is a build-time only dependency with no
runtime behavior.

**Recommendation**: Migrate from `pqcrypto-mldsa` to an alternative
ML-DSA-65 implementation before tagging v0.9-rc1. Candidate
replacements:
1. `aws-lc-rs` (already in the dependency tree for TLS) — provides
   ML-DSA via AWS-LC C library
2. `fips204` crate (pure Rust ML-DSA implementation)
3. Direct binding to NIST reference implementation

### 8.4 Critical Finding: pqcrypto Unmaintained

The `pqcrypto` family (`pqcrypto-mldsa`, `pqcrypto-internals`,
`pqcrypto-traits`) is the ML-DSA-65 implementation used for:
- Agent identity key generation
- AgentRecord signing and verification
- Handshake signature verification

These crates wrap the PQClean C reference implementation. The
PQClean project is being archived (announced 2026-06-04), meaning:
1. No security patches will be issued
2. No bug fixes will be made
3. No FIPS compliance updates will be made
4. The crates will remain on crates.io but will not be maintained

**Risk assessment**: HIGH. A protocol whose primary authentication
mechanism depends on unmaintained cryptographic software is not
suitable for production deployment.

**Mitigation options** (in priority order):
1. **Migrate to `aws-lc-rs`**: Already in the dependency tree (used
   by rustls). AWS-LC provides ML-DSA-65 via a maintained C library
   with FIPS validation pathway.
2. **Migrate to `fips204`**: Pure Rust implementation of FIPS 204.
   Smaller dependency footprint but less battle-tested.
3. **Vendor the PQClean code**: Fork the C code and maintain it
   internally. Highest maintenance burden but maximum control.

This migration should be completed before tagging v0.9-rc1.
