# AAFP Golden Wire Traces

This directory contains permanent regression fixtures capturing
complete AAFP v1 protocol exchanges. Each trace is a directory
containing:

- `trace.bin`: Raw bytes of the entire exchange
- `trace.hex`: Hex dump with frame boundaries marked, decoded fields,
  and annotations
- `meta.json`: Machine-readable metadata describing frames, transcript
  hashes, session ID (if established), and final protocol outcome

## Verification

Traces are generated by the Rust reference implementation and verified
by the Go independent implementation. The Go verifier checks:

1. **Frame boundaries**: Each frame decodes at the expected offset
   with the expected length.
2. **Frame fields**: Type, flags, stream ID, payload length, and
   extension length match the metadata.
3. **Transcript hashes**: Recomputed from the handshake message bytes
   using the same TLS binding value, matching the metadata.
4. **Session ID**: Present and valid hex when the handshake succeeds.
5. **Total bytes**: The sum of all frame lengths equals the total
   trace size.

## Deterministic Inputs

All traces use deterministic inputs to ensure reproducibility:

- TLS channel binding: `0xAA` repeated 32 times
- Client nonce: `0x01` repeated 32 times
- Server nonce: `0x02` repeated 32 times
- Client public key: deterministic 1952-byte pattern
- Server public key: deterministic 1952-byte pattern
- Signatures: deterministic 3309-byte patterns (not real ML-DSA-65
  signatures — these traces test wire format, not signature
  verification)

## Usage

### Generating traces (Rust)

```bash
cargo run --bin generate_golden_traces -- /path/to/golden_traces
```

### Verifying traces (Go)

```bash
go test ./goldentrace/ -v
```

## Future Traces

Additional traces to capture as the implementation matures:

- Invalid transcript hash (requires live handshake with real crypto)
- Expired AgentRecord
- Mismatched key_algorithm
- Multiple concurrent streams
- Fragmented DATA frames (MORE flag)
- Compressed payload
- PING/PONG keepalive exchange
- Non-critical unknown frame type (skip behavior)
- Discovery lookup request/response
